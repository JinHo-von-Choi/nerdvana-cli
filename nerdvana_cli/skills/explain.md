# Explain Protocol

When explaining code:

1. Start with the **purpose** — what problem does this solve?
2. Show the **data flow** — inputs -> processing -> outputs
3. Highlight **key decisions** — why was it done this way?
4. Note **gotchas** — non-obvious behavior, edge cases
5. Suggest **further reading** — related files, docs, patterns

Keep language simple. Use diagrams (ASCII) when helpful.
Tailor depth to the question — don't over-explain obvious parts.
