## 예상 못한 이슈

### 1. 순환 import 8건 선재 존재
로드맵 v3.1은 "cycle 0"을 exit criterion으로 명시했으나 0.4.1 코드에 이미 8개 순환 존재.
해결: T-0A-01을 baseline 모드로 전환. `.import_cycles_baseline.json`에 현 8개 기록, delta > 0만 실패 처리.

주요 순환:
- `commands.model_commands ↔ ui.app`
- `tools.registry ↔ tools.agent_tool`
- `core.subagent ↔ core.agent_loop`
- `core.lsp_client ↔ tools.lsp`
- 기타 4건.

`core.subagent ↔ core.agent_loop`는 분해 후 자연 해소되지 않았다. baseline에 남겨둠. Phase 0B 이전 별도 순환 정리 티켓(`T-debt-import-cycles`) 권장.

### 2. ralph_loop_check 버그 발견
T-0A-02 에이전트가 "end_turn stop_reason에서 AFTER_API_CALL 이벤트가 emit되지 않아 ralph_loop_check 훅이 실제 fire되지 않는" 0.4.1 버그 포착.
방침(옵션 E): Phase 0A는 구조 변경만. 버그는 스냅샷 베이스라인으로 동결. 별도 티켓 `T-bug-ralph-loop` 권장.

### 3. 테스트 증분 목표 미달
목표 +25, 실제 +20 (8 parity + 9 contracts + 3 fault-injection). 차이 5건은 contract 테스트의 범위가 초기 계획보다 소박. 큰 리스크 없음.

### 4. 기존 테스트 파일 일부 수정 필요
`_is_retryable_error` 모듈 레벨 함수가 T-0A-05에서 `LoopHookEngine`으로 이전되면서 `tests/test_model_fallback.py`가 깨졌다. 에이전트가 `LoopHookEngine()._is_retryable_error` 호출로 수정. 외부 사용자 API 변경은 아니므로 SemVer patch 호환 유지.

### 5. tests/ 외부 pre-existing ruff 위반
`tests/`와 `examples/`에 이미 위반이 다수 있었다. `ruff check .`은 실패하지만 `ruff check nerdvana_cli/`는 0 위반. 본 릴리즈에서는 production 코드만 게이팅.
