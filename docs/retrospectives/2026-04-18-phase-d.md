## 2. 아키텍처 결정

### 2.1 NamePathResolver — `Foo/bar.baz` 문법

슬래시(`/`) 구분자로 계층을 표현하고, 점(`.`)은 모듈 경로 내에서만 허용한다.
단일 regex `[A-Za-z_][A-Za-z0-9_.]*` 로 각 세그먼트를 검증함.
workspace-wide symbol 검색 (`within=None`) 은 0.5.0에서 구현 보류.
현재 `LspClient` 는 `workspace/symbol` 요청을 지원하지 않으므로 
`within=` 필수로 명시하고 `LspSymbolError` 로 안내.

### 2.2 CodeEditor — TTL 없음, LRU 20개

로드맵 v3 §0.0.4 반영: TTL 60s는 ultrawork/compaction 조합에서 취약.
`OrderedDict` + `move_to_end` 패턴으로 LRU 구현.
`evict_lru(max_count)` 메서드명: `max`는 파이썬 builtin 섀도잉 위반(ruff A002).

### 2.3 diff 생성 분리

`create_preview` 는 `new_contents` (경로 → 제안 내용)만 받아 diff 생성.
실제 파일 쓰기는 `workspace_edit` 경로 (LSP `_apply_workspace_edit`)로 분리.
이 구조 덕분에 CodeEditor 단위 테스트가 LspClient 없이 실행됨.

### 2.4 `_find_symbol_end` — 인덴트 기반 본문 감지

Python 소스에서 def/class 헤더 이후 인덴트가 감소하는 지점을 본문 끝으로 판단.
빈 줄은 건너뜀. 엄밀한 AST 파싱 없이 간단하게 처리.
0.5.1에서 Tree-sitter 또는 python AST 기반으로 개선 예정.

## 3. 미해결 부채 (0.5.1 이후)

1. **workspace-wide FindSymbol**: `workspace/symbol` LSP 요청 미구현.
   현재 `within=` 없으면 `LspSymbolError`. 0.5.1 대상.
2. **include_body=True**: `FindSymbol` 에서 심볼 본문 텍스트 반환 미구현.
   0.5.1 InsertBefore/InsertAfter/Delete/Rename 도구와 함께 추가.
3. **`_find_symbol_end` AST 정확도**: 인덴트 기반 휴리스틱. TypeScript 지원 불가.
   0.5.1에서 LSP `selectionRange` 활용으로 전환.
4. **ralph-loop 버그**: Phase 0B 부채 그대로 보존 (건드리지 않음).
5. **import cycles**: baseline 8개 유지 (Phase 0A/0B 부채 그대로).
