# Phase G1 Retrospective — MCP Server Mode + Auth + ACL + Audit Log

**Author**: 최진호
**Date**: 2026-04-18
**Version**: 0.8.0

## Summary

Phase G1 adds a production-ready MCP 1.0 server to nerdvana-cli, enabling
external harnesses (Claude Code, Cursor, Continue, etc.) to call nerdvana tools
via the `mcp__nerdvana__*` namespace over either stdio or HTTP JSON-RPC.

### CLI Commands Added

- `nerdvana serve --transport {stdio,http} [--port] [--host] [--allow-write] [--tls-cert] [--tls-ca]`
- `nerdvana admin acl list`
- `nerdvana admin acl revoke <prefix>`
- `nerdvana admin acl add <name> <roles>`

### Authentication Layers (3 of 3 implemented)

1. HTTP Bearer — sha256 hash match against `~/.nerdvana/mcp_keys.yml`
2. stdio / Unix socket — `0600` permission + UID equality check
3. mTLS — peer certificate CN; unknown CN falls back to `read-only`

### ACL Design

Default role `read-only` grants 6 tools:
`symbol_overview`, `find_symbol`, `find_referencing_symbols`,
`ReadMemory`, `ListMemories`, `GetCurrentConfig`.

Roles `edit`, `write-memory`, `admin` unlock write tools.
Unknown clients receive `read-only` automatically (v3.1 §3.1).

### Write-Tool Dual Gate (v3 §7.5)

Write tools require both:
1. `--allow-write` runtime flag
2. `confirm: true` field in the request JSON

Either gate missing → immediate PermissionError.

### Audit Log

- SQLite WAL mode, file `~/.nerdvana/audit.sqlite`, permissions `0600`
- Every call records: ts, client_identity, transport, tool_name, args_hash, decision, duration_ms, error_class
- Pruning: after every 1 000 writes, if file > 1 MB, oldest 25 % of rows deleted

  Phase G2 will wire real tool execution via the existing ToolRegistry.
- `hook_bridge.py` dispatch is a no-op placeholder.

- mTLS cert loading is accepted by the CLI but not yet passed to the TLS layer.
